kv-adx-access (1ci-preprod-metrics)
Data Protection & Recovery - Critical: Soft delete not enabled
Impact: Permanent data loss if secrets/certificates/keys are accidentally deleted - no recovery possible
Best Practice: Soft delete must be enabled on all Azure Key Vaults to protect against accidental deletion. This provides a 7-90 day recovery window for deleted secrets.
Recommendation: Enable soft delete protection (mandatory for production vaults)
üîß Remediation Steps
  1. Enable soft delete: Update-AzKeyVault -VaultName -EnableSoftDelete
  2. Configure retention period (7-90 days, default is 90)
  3. Enable purge protection for additional security: -EnablePurgeProtection
  4. Test recovery by deleting and restoring a test secret

üìö Documentation: https://docs.microsoft.com/en-us/azure/key-vault/general/soft-delete-overview

kv-azure2cyberark (1ci-prod-metrics)
Network Security - Medium: Public network access enabled without private endpoints
Impact: Potential exposure to public internet attacks, data exfiltration risks
Best Practice: Azure Key Vault should be configured with private endpoints to ensure traffic remains on the Azure backbone network and is not exposed to the public internet.
Recommendation: Configure private endpoints or restrict network access with firewall rules
üîß Remediation Steps
  1. Create a private endpoint for the Key Vault in your virtual network
  2. Configure private DNS zone for privatelink.vaultcore.azure.net
  3. Update Key Vault firewall to deny public access: Set-AzKeyVaultAccessFirewall -VaultName -DefaultAction Deny
  4. Allow access from specific virtual networks or IP ranges if needed
  5. Test connectivity from authorized networks only

üìö Documentation: https://docs.microsoft.com/en-us/azure/key-vault/general/private-link-service
